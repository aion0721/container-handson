# 21. k3s を使って簡易的に起動する

![ページのイメージ図](/illustrations/kubernetes.svg)


## この章でやること

![k3s で Pod と Service を作る流れ](/flows/21_k3s.svg)

次は Kubernetes 環境である `k3s` を使って、同じようなアプリをデプロイします。

## 21-1. Pod を作る

まずは動作確認として、`httpd` の Pod を直接作成します。

```bash
kubectl run httpd --image=docker.io/library/httpd:2.4 --port=80
```

実行結果例:

```bash
pod/httpd created
```

作成されたリソースを確認します。

```bash
kubectl get all
```

実行結果例:

```bash
NAME       READY   STATUS    RESTARTS   AGE
pod/httpd  1/1     Running   0          20s

NAME                 TYPE        CLUSTER-IP   EXTERNAL-IP   PORT(S)   AGE
service/kubernetes   ClusterIP   10.43.0.1    <none>        443/TCP   1d
```

## 21-2. Service を作る

Pod にアクセスできるように Service を作成します。

```bash
kubectl expose pod httpd --type=NodePort --port=80
```

実行結果例:

```bash
service/httpd exposed
```

作成されたリソースを確認します。

```bash
kubectl get all
```

実行結果例:

```bash
NAME       READY   STATUS    RESTARTS   AGE
pod/httpd  1/1     Running   0          1m

NAME                 TYPE        CLUSTER-IP      EXTERNAL-IP   PORT(S)        AGE
service/httpd         NodePort    10.43.122.210   <none>        80:31234/TCP   10s
service/kubernetes   ClusterIP   10.43.0.1       <none>        443/TCP        1d
```

NodePort が割り当てられているので、そのポートにアクセスします。

```bash
curl http://localhost:<NodePort>
```

::: tip
`<NodePort>` の部分は `kubectl get all` の `service/httpd` に表示されたポート番号を見て置き換えてください。
:::

## 21-3. 後片付け

```bash
kubectl delete svc httpd
kubectl delete pod httpd
```
