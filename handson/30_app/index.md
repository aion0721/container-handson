# 30. 発展編

![ページのイメージ図](/illustrations/apps.svg)


この章は、本編が早く終わった人向けの発展コンテンツです。

本編では `httpd` イメージを使って、コンテナと Kubernetes の基本を学びました。
発展編では、その知識を **自分たちのアプリにどう使うか** に寄せて考えます。

たとえば、ハッカソンで作ったアプリをチームメンバーの環境でも動かしたいときや、Java 研修で作るアプリをコンテナ化したいときに、ここで扱う考え方が役に立ちます。

::: tip
4 時間のハンズオンでは、まず 10. Podman 編と 20. Kubernetes 編を優先します。
30. 発展編は、時間に余裕がある場合や自習用として取り組んでください。
:::

## セクション

31. [簡単なアプリ](./31_simple_app)

    HTML だけの小さなアプリをコンテナ化し、`k3s` で動かします。
    「自分で作ったものをコンテナとして配布・起動する」流れをつかみたい人向けです。

32. [Java のアプリ](./32_java_app)

    Spring Boot アプリを JAR にして、実行用イメージを作り、`k3s` にデプロイします。
    このあとの Java 研修や、ハッカソンで作る API サーバのコンテナ化に近い内容です。

33. [開発環境として利用する](./33_dev_env)

    コンテナをアプリの実行環境だけでなく、開発環境として使います。
    チーム内で Java やツールのバージョン差を減らしたいときのヒントになります。
