# Podman と k3s ではじめるコンテナハンズオン

![ページのイメージ図](/illustrations/overview.svg)


## はじめに

このハンズオンでは、`podman` を使ってコンテナの基本操作を体験し、その後 `k3s` を使って Kubernetes 上でアプリケーションを動かします。

本編では `httpd` イメージを使いながら、以下の流れで進めます。

1. コンテナイメージの取得・削除
2. コンテナの起動・停止・削除
3. コンテナイメージのビルド・修正
4. コンテナのメリットを体験
5. k3s を使った簡易デプロイ
6. Manifest でのデプロイ
7. Service と Ingress での公開
8. クラスターのメリットを体験

時間に余裕がある場合や、さらに進めたい場合は、発展編としてアプリのコンテナ化にも挑戦できます。

- 簡単なアプリを起動
- Java のアプリをコンテナ化してデプロイ
- コンテナを開発環境として利用

::: tip
30. 発展編は、本編が早く終わった人向けの追加コンテンツです。
このあとの Java 研修やハッカソンで、アプリをコンテナ化するときのヒントとしても使えます。
:::

## ゴール

このハンズオンを終えると、次のことができるようになります。

- コンテナイメージを取得・確認・削除できる
- コンテナを起動・停止・削除できる
- `Containerfile` を使って独自イメージを作成できる
- コンテナの再現性・分離・使い捨てやすさを確認できる
- Kubernetes にアプリをデプロイできる
- Deployment による Pod の自然復旧を確認できる
- Service と Ingress を使ってアプリを公開できる
- スケールやローリングアップデートを体験できる

## 前提条件

- Linux 環境、または Linux コマンドが使える環境
- `podman` がインストール済みであること
- `k3s` が利用可能であること
- `curl` またはブラウザで動作確認できること

::: tip
コマンドは例として記載しています。環境によって `sudo` が必要な場合があります。
:::
