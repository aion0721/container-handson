# Podman と k3s ではじめるコンテナハンズオン

![ページのイメージ図](/illustrations/overview.svg)

## はじめに

このハンズオンでは、`podman` を使ってコンテナの基本操作を体験し、その後 `k3s` を使って Kubernetes 上でアプリケーションを動かします。

本編では `httpd` イメージを使いながら、以下の流れで進めます。

1. コンテナイメージの取得・削除
2. コンテナの起動・停止・削除
3. コンテナイメージのビルド・修正
4. コンテナのメリットを体験
5. k3s を使った簡易デプロイ
6. Manifest でのデプロイ
7. Service と Ingress での公開
8. クラスターのメリットを体験

時間に余裕がある場合や、さらに進めたい場合は、発展編としてアプリのコンテナ化にも挑戦できます。

- 簡単なアプリを起動
- Java のアプリをコンテナ化してデプロイ
- コンテナを開発環境として利用

::: tip 30. 発展編は、本編が早く終わった人向けの追加コンテンツです。
このあとの Java 研修やハッカソンで、アプリをコンテナ化するときのヒントとしても使えます。
:::

## ゴール

このハンズオンを終えると、次のことができるようになります。

- コンテナイメージを取得・確認・削除できる
- コンテナを起動・停止・削除できる
- `Containerfile` を使って独自イメージを作成できる
- コンテナの再現性・分離・使い捨てやすさを確認できる
- Kubernetes にアプリをデプロイできる
- Deployment による Pod の自然復旧を確認できる
- Service と Ingress を使ってアプリを公開できる
- スケールやローリングアップデートを体験できる

## 前提条件

このハンズオンでは、Linux 環境に接続して作業する方法として、主に次の 2 パターンを想定しています。

![Linux 環境への接続パターン](/illustrations/linux-connection-patterns.svg)

- パターン A: 自分の端末から Linux などのサーバーに SSH 接続し、その Linux 上で作業する
- パターン B: 自分の端末自体に Linux 環境を用意し、WSL などの上で作業する

- Linux 環境、または Linux コマンドが使える環境
- `podman` がインストール済みであること
- `k3s` が利用可能であること
- `curl` またはブラウザで動作確認できること
- ファイルを編集できるエディタが使えること

Windows で WSL を使う場合は、PowerShell を開いて `wsl` コマンドを実行し、WSL の Linux 環境に入ってから準備やハンズオンのコマンドを実行してください。

このハンズオンでは、サーバー上での作業に慣れるために `vim` の操作も少し扱います。
すでに `vim` が使える人はそのまま進めてください。
`emacs` や `nano` など別のエディタを使いたい人は、それを使って構いません。
エディタ操作が分からない場合は、[Vim の最低限操作](/guide/vim) を確認してください。

::: tip
コマンドは例として記載しています。環境によって `sudo` が必要な場合があります。
:::

::: warning
同じ k3s クラスターを複数人で共有する場合は、受講者ごとに Namespace や NodePort を分けてください。
本資料では、1 人 1 環境で進める前提で一部の NodePort を固定しています。
:::

::: tip
以下のコマンドを実行しておくと `Tab` 補完がされるので、非常に操作しやすくなります。

```bash
source <(kubectl completion bash)
```
:::
