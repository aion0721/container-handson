# コンテナとは

コンテナは、アプリケーションとその実行に必要な依存関係をひとまとめにした実行単位です。

## 何がうれしいのか

- **環境差分を減らせる**: 開発環境と本番環境で同じイメージを使える
- **起動が速い**: VM より軽量で、短時間で起動しやすい
- **配布しやすい**: レジストリ経由でイメージを共有できる

## イメージとコンテナの違い

- **イメージ**: アプリ実行に必要なファイルをまとめた「設計図」
- **コンテナ**: イメージから起動した「実行中の実体」

## 基本的なライフサイクル

1. イメージを取得またはビルドする
2. イメージからコンテナを起動する
3. 動作確認する
4. 停止・削除する

実際の操作は [Hands-on](/handson/) の Podman 編で体験できます。
